# Mock momoshop — Design Notes (Phase 1)

> 這是 Phase 1 的設計筆記。先確立架構方向、取捨與 Agent 協作策略，再進入實作。
> 評估者請從這裡開始閱讀；實作後的最終總結見根目錄 `README.md`。

## 題目選擇

選 **A. Mocking momoshop**：純前端、無真實 API，重建 momo 購物網的核心體驗。

## 核心判斷：為什麼是「窄而深」而不是「廣而淺」

題目反覆強調「不只功能完成度」「請說明哪些優先、哪些刻意不做」「把自己當長期維護者」。
因此本專案的目標不是還原最多頁面，而是證明三件事：

1. **架構是否能長期演進** —— 卡片、資料層、路由是否乾淨可擴充。
2. **取捨是否有意識** —— 哪些刻意不做、為什麼，寫在 `02-tradeoffs.md`。
3. **能否工程化地指揮 AI** —— commit 顆粒度、驗收、防護，見 `03-agent-collaboration.md`。

## 對真實 momo 的觀察（決定「神韻」）

實際檢視 `momoshop.com.tw`（品牌色 `#D62872` 洋紅、版本 `v2606.x`）後，抓出最具識別度的特徵：

- **高資訊密度**：台式電商「商品牆」，一屏塞滿卡片，不走留白風。
- **促銷驅動**：限時下殺（倒數）、每日下殺、折價券、mo幣、滿額折、免運、N 折 badge。
- **多種商品卡變體**：標準卡、限時下殺卡（倒數 + 售出進度）、排行榜卡（名次）、直播/電視購物卡。
- **首頁節奏**：輪播 banner → 分類導覽 → 限時下殺 → 多組主題商品列 → 排行榜。

> 結論：抓到「密度 + 促銷卡變體 + 洋紅色系」即可傳達 momo 神韻，不追求像素級還原。
